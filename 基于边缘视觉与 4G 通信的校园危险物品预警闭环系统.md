# 危险物品检测预防系统 — 项目总结

## 一、项目概述

本项目面向校园教室、走廊等公共安全场景，设计了一套基于 STM32、OpenMV H7 Pro 和 L610 4G 模块的危险物品检测与预防系统。系统并不单纯追求危险物品识别精度，而是将视觉识别作为安全预防流程的入口：当 OpenMV 发现疑似危险物品后，STM32 控制云台完成目标锁定与稳定追踪，随后进行数字裁剪放大和二次确认；确认风险后，系统自动截图，并通过 L610 模块完成 MQTT 报警和 HTTP 图片上传，将报警信息与图片链接推送至手机端，供人工复查。

项目重点从“识别危险物品”转向“识别后的预防处置流程”，形成“检测—锁定—确认—报警—复查”的闭环机制，从而提升校园场景下危险物品发现、预警和人工干预的效率。

## 二、系统架构

整体采用 **"边缘视觉计算 + 嵌入式AI决策"** 的分层架构，设计思路为：

```
智能云台 + 危险物品检测 + 实时报警
```

## 三、核心技术方案

### 1. 图像采集与识别（OpenMV H7 Pro）

- 使用 OpenMV H7 Pro 进行图像采集与目标识别
- 通过训练危险物品数据集，实现对危险物品的自动识别
- **难点**：在教室、校园等实际场景中，人多、背景复杂、目标物体较小且距离较远，识别成功率会显著下降

### 2. 云台控制（STM32 + PID）

- OpenMV 识别目标后，通过计算 **偏差值**，经串口发送给 STM32
- STM32 接收偏差后进行 **PID 运算**（建议不要一上来就用完整 PID），输出 PWM 信号控制舵机
- 实现云台对目标的自动追踪

### 3. 目标确认与截图

- 云台稳定后，OpenMV 对识别的物品进行 **数字裁剪放大**
- 二次确认是否为危险物品，确认后对放大物品进行截图

### 4. 远程报警与图片上传（广和通 L610 4G 模块）

- OpenMV 通过串口将照片数据发送给 L610，同时发送 AT 指令和 MQTT Topic
- L610 收到 AT 指令后启动 MQTT，将 Topic 转发至 Broker，手机端订阅对应主题实现 **实时报警**
- L610 将照片通过 HTTP 上传至免费图床，图床 **自动原路返回图片链接**，L610 硬件自动接收
- 再通过 MQTT 将图片链接发送至手机端，供 **人工复查**

## 四、数据流总结

```
OpenMV 图像采集 → 训练数据集识别危险物品
    → 计算偏差 → 串口发送偏差值 → STM32 PID运算 → PWM控制舵机追踪
    → 云台稳定 → 数字裁剪放大 → 二次确认
    → 截图 → 串口发送照片+AT指令+Topic → L610 4G模块
    → MQTT 推送报警 → 手机端订阅接收
    → HTTP 上传图片至图床 → 图床返回链接 → L610接收
    → MQTT 发送图片链接 → 手机端查看进行人工复查
```

## 五、关键器件

| 模块 | 型号 | 作用 |
|------|------|------|
| 主控 | STM32 | 核心控制、PID运算、舵机PWM驱动 |
| 视觉 | OpenMV H7 Pro | 图像采集、AI识别、偏差计算、截图 |
| 通信 | 广和通 L610 | 4G联网、MQTT报警、HTTP图片上传 |
